# 口播脚本生成工作流程

AI 根据原始内容和幻灯片生成口播脚本时的详细指南。

## 概要

**目的**: 将幻灯片内容扩展为适合口播的详细叙述脚本

**原则**:
1. **补充背景**: 幻灯片是简洁的，脚本需要补充背景知识和过渡
2. **口语化**: 使用自然口语表达，避免书面语
3. **节奏控制**: 标注各部分预估时长
4. **情感注入**: 加入适当的情感表达和强调
5. **忠实原始**: 不添加原始内容未记载的信息
6. **视频对齐**: 脚本分段应能映射到连续的幻灯片范围，方便后续音频和视频时间线同步

---

## 工作流程

### Step 1: 读取文件

```
1. 读取 weekly/{episode}/source.md (原始内容)
2. 读取 weekly/{episode}/slides.md (生成的幻灯片)
3. 理解整体叙事结构
```

### Step 2: 确定脚本结构

#### 标准结构模板

```md
# {项目名} 口播脚本（约 X 分钟）

---

## 一、开场（约 1-2 分钟）

[开场白 + 项目引入]

---

## 二、[第一部分标题]（约 X 分钟）

[详细内容]

---

## 三、[第二部分标题]（约 X 分钟）

[详细内容]

...

---

## X、结尾（约 1 分钟）

[总结 + 引导]

---
```

### Step 3: 各部分生成规则

#### 开场

**内容**:
- 固定开场白："大家好，欢迎来到本期 weeko 周报。"
- 项目引入：用一个问题或场景引起兴趣
- 简要预告本期内容结构

**风格**: 轻松、引起好奇心

---

#### 主体部分

**转换规则**:

| 幻灯片类型 | 脚本处理 |
|-----------|---------|
| 标题/分隔 | 作为章节标题，补充过渡语 |
| 列表 | 展开每项内容，补充说明 |
| 表格 | 口述关键对比点，不逐行读 |
| 代码 | 描述代码作用，不逐行读 |
| 图片 | 描述图片内容和意义 |
| 功能展示 | 详细描述功能价值和应用场景 |

**扩展技巧**:
- 添加"为什么这个重要"的解释
- 补充实际应用场景
- 加入过渡语连接各部分
- 使用"我们来看..."、"值得注意的是..."等口语表达

#### 与视频时间线对齐

每个 `##` 分段会成为 TTS 的一个音频片段，并进一步成为视频时间线的一个大段。因此脚本结构应满足：

- 分段顺序与 `slides.md` 的叙事顺序一致。
- 一个脚本分段对应一组连续幻灯片，避免跨段反复跳页。
- 列表或 `<v-clicks>` 内容按出现顺序口播，方便视频中逐个 click state 对齐。
- 每段标题尽量稳定，因为音频文件名会使用标题生成。

---

#### 结尾

**内容**:
- 核心价值总结（3-5 点）
- 项目链接提醒
- 固定结尾："感谢观看，这里是 weeko 周报，我们每周精选有价值的技术项目，下期见。"

---

## 风格指南

### 口语化表达

| 书面语 | 口语化 |
|-------|-------|
| "它是..." | "这个项目是..." |
| "具有以下特点" | "它有这么几个特点" |
| "需要注意的是" | "值得注意的是" |
| "请参考" | "可以去看看" |

### 情感注入

- **强调**: 用"**重点**"、"核心"、"关键"等词
- **惊讶**: 适当使用"有意思的是"、"你可能会惊讶"
- **引导**: 用"我们来看看..."、"接下来..."

### 节奏控制

- 每部分标注预估时长
- 复杂内容放慢节奏
- 简单概览加快节奏
- 建议每部分最多 3-5 分钟

---

## 注意事项

### 不应做的事

1. **过度扩展**: 不添加原始内容未记载的信息
2. **逐行朗读**: 表格、代码不逐行读
3. **书面化**: 避免论文式表达
4. **失去结构**: 脚本结构应与幻灯片对应

### 必须做的事

1. **保持忠实**: 使用原始内容的信息
2. **口语化**: 自然的讲演风格
3. **标注时长**: 每部分预估时间
4. **完整覆盖**: 所有幻灯片内容都应在脚本中出现

## 生成示例

### 输入: 幻灯片片段

```md
# 独特之处

<v-clicks>

- **闭环学习** — Agent 自动策展记忆、创建技能、自我改进、跨会话召回
- **Web 应用界面** — 轻量 WebUI，与 CLI 功能完全一致

</v-clicks>
```

### 输出: 脚本对应部分

```md
## 三、独特之处（约 4 分钟）

Hermes 有六个核心特性，我们逐一来看。

**第一个：闭环学习。**

这是 Hermes 最核心的差异化能力。Agent 会自己策展记忆，
自动创建技能，这些技能在后续使用中会自我改进，并且可以
跨会话召回。整个过程形成了一个完整的闭环：经历到提炼，
到技能，到应用，再到优化。不需要你手动告诉它该学什么，
它自己会判断什么值得记住。

**第二个：Web 应用界面。**

Hermes WebUI 是一个适用于 Hermes Agent 的轻量级 Web 应用界面。
与 CLI 体验完全一致——你可以在终端中执行的所有操作，
都可以在此界面中完成。无需构建步骤，无需框架，无需打包工具。
仅使用 Python 和原生 JS。
```
